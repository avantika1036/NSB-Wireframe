# NSB Website — Mid-Fidelity Wireframe Prototype

Interactive wireframe prototype for the **Network Simulation Bridge (NSB)** website redesign, created as part of a GSoC 2026 proposal under **UC Santa Cruz OSPO**.

## 🔗 Live Preview

[avantika1036.github.io/NSB-Wireframe](https://avantika1036.github.io/NSB-Wireframe/)

## About This Prototype

This is a mid-fidelity, fully interactive wireframe covering all 7 key screens of the proposed NSB website. It was built to demonstrate the information architecture, user flow, and onboarding strategy before moving to high-fidelity design.

The primary design goal is to get a new user from the home page to a **working simulation in under 5 minutes**.

## Screens Included

| # | Screen | Purpose |
|---|--------|---------|
| 0 | User Flow | End-to-end journey map for all three user types |
| 1 | Home | Hero, feature strip, terminal preview |
| 2 | Get Started | OS tab switcher, step-by-step install guide |
| 3 | Quickstart | Copy-paste steps, First Success checkpoint |
| 4 | Documentation | Sidebar nav, config reference, API reference |
| 5 | Tutorials | Difficulty-filtered card grid |
| 6 | Contribute | Three contributor paths, Good First Issues |
| 7 | Community / About | Project info, team, communication channels |

## Design Principles

- **First Success First** — user runs a working simulation before reading any docs
- **Progressive Disclosure** — advanced concepts appear only after initial setup
- **Clear Verification** — every step shows expected output
- **Separation of Concerns** — quickstart and reference docs are fully separate

## User Journeys
```
New User:      Home → Get Started → Quickstart → ★ First Success → Docs/Tutorials → Contribute
Intermediate:  Docs → Config Reference → API Reference
Contributor:   Contribute → Dev Setup → Architecture → First Issues → First PR
```
